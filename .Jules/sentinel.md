The codebase originally suffered from Stored Cross-Site Scripting (XSS) due to insecure injection of variables directly into HTML file reads. This was fixed by using the python `html` library and applying `html.escape()` dynamically on all dynamically set properties (including `{{device_id}}`, `{{question}}`, `{{name}}`, `{{unique_code}}`, `{{credits}}`, and `{{token}}`). It is imperative that all properties set by the user remain escaped when displayed in any HTML structure to prevent script injections.
